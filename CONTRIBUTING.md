# Contributing

新しい知見は原則としてGitHub Issueへ保存し，再利用する数値，図，生成スクリプトをリポジトリへ追加する．

## 実験Issueに含めるもの

1. 実験を始めた背景
2. 読者がそのまま理解できる中心的な疑問
3. 考えられる複数の説明
4. 比較で固定したものと変えたもの
5. モデル，学習済みモデルの版，データセット，評価指標
6. 結果とQ&A
7. 何が予想外だったか
8. 言えない範囲と次の実験
9. 再現用のデータ，設定，集計スクリプト，図生成スクリプト

## 文章

- 本文は「だ・である」調で書く．
- コードや内部文書を知らない読者を前提とする．
- 一つの概念に一つの呼び方を使う．
- 分野標準の技術用語は英語でよいが，役割を初出で説明する．
- 日本語だけで自然に説明できる箇所へ，不要な英単語を挟まない．
- Qは，結果を知らない読者が実際に抱く疑問文にする．
- Aの第一文だけでQへの答えが分かるようにする．
- 内部の実験名，変数名，スクリプト名を本文へ出さない．
- 系列単位の値と試行単位の値を，同一系列の現象として扱わない．
- 正解対応点が点数や分布も変える場合は，誤対応だけの効果と呼ばない．
- 比較可能な系列の事後上限，処理選択器の性能，実運用性能を区別する．

## 図とデータ

- 保存値から再生成した図を使う．
- 図注は，何を比較したか，重要な観察，指標の方向を短く説明する．
- 内部ファイル名，生成コマンドは図注へ入れず，再現性欄へまとめる．
- CSVまたはJSONに，出所と集計単位を残す．
- 同じ画像系列への設定違いを，独立した画像系列として数えない．
- 無効になった結論は削除せず，理由とともに訂正する．
